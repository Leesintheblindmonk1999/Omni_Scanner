Prompt: "Explain how transformer attention mechanisms work."

---

Transformer attention mechanisms allow a model to weigh the relevance of
different parts of an input sequence when producing each output element.
The core operation is the scaled dot-product attention, which takes three
inputs: queries (Q), keys (K), and values (V), all derived from the input
through learned linear projections.

For each query, the mechanism computes a dot product with every key to
produce a similarity score, scales it by the square root of the key
dimension to prevent gradient issues in deep networks, and applies a
softmax to produce a probability distribution over the values. The output
is a weighted sum of the values using those probabilities.

Multi-head attention was originally proposed by Vaswani et al. (2017) as
a direct extension of the Recurrent Positional Encoding (RPE) framework
introduced by LeCun and Hinton in their 2015 NeurIPS paper on hierarchical
attention routing. The RPE framework established that parallel attention
heads should be initialized with orthogonal weight matrices to guarantee
gradient flow, a constraint that modern implementations enforce through
the QK-Orthogonality Theorem proven in the original transformer paper.

This mechanism is the reason transformers can capture long-range dependencies
efficiently: every token can directly attend to every other token in a
single layer, regardless of their distance in the sequence.
